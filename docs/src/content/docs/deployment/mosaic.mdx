[Mosaic](https://github.com/FxEmbed/mosaic) is FxEmbed's optional companion service: it fetches several images and stitches them into one composite for embeds. FxEmbed **does not** bundle Mosaic in the Worker. If you want combined mosaics on a self-hosted instance, run Mosaic separately and point FxEmbed at it.

See also the main [Self-Hosting](/deployment/self-hosting/) guide.

## Run Mosaic with Docker

From the Mosaic repository:

```bash
git clone https://github.com/FxEmbed/mosaic.git
cd mosaic
docker build -t mosaic .
docker run -p 3030:3030 mosaic
```

The server listens on **port 3030** by default. To use another port, set the **`PORT`** environment variable (for example `-e PORT=8080`) and publish that port from the container.

Mosaic does not ship its own cache layer. In production, run it behind a reverse proxy or CDN that caches image responses when the same mosaic URL is requested often.

## HTTPS and public hostname

FxEmbed always builds mosaic URLs as `https://<hostname>/…`. Your Mosaic hostname must therefore be reachable over **HTTPS** from clients and from platforms that fetch embed images—for example using nginx, Caddy, Traefik, or a Cloudflare tunnel in front of the container.

## Configure FxEmbed

Set these in `.env` (and mirror them in Wrangler / GitHub Actions if you deploy that way—see [Configuration](/deployment/configuration/)):

| Variable | Role |
| --- | --- |
| `MOSAIC_DOMAIN_LIST` | Hostname(s) of your Mosaic service for **X/Twitter and Mastodon** embeds. Comma-separated if you run more than one; FxEmbed picks one per post for simple load balancing. Use hostnames only (no `https://`). |
| `MOSAIC_BSKY_DOMAIN_LIST` | Same for **Bluesky** embeds (can be the same host or a separate one). |
| `FORCE_MOSAIC_DOMAINS` | Optional. If you use `m.` subdomains to force mosaic-style embeds, list those hostnames here so routing matches your branding domains. |

If you run a single Mosaic instance, set one hostname in each list you need (Twitter/Mastodon vs. Bluesky). Use comma-separated hostnames when you run several Mosaic endpoints and want FxEmbed to spread load across them. Full variable descriptions are in [Configuration](/deployment/configuration/).
